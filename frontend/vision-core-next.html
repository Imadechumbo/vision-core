DIFERENCIAL

Copiloto com sistema adversarial.

Outros copilotos pedem ao mesmo modelo que criou o erro para verificar se o erro foi corrigido. Nós não.

Verificadores matemáticos determinísticos independentes confirmam a correção antes de qualquer avaliação por linguagem natural. O LLM é a última voz — nunca a única.
Verificação Determinística Antes de qualquer LLM opinar, verificadores matemáticos proprietários confirmam a correção. Não interpretam. Não opinam. Não alucinam.
Agente Auditor Separado O sistema que cria o patch nunca é o mesmo que o valida. Um agente especializado com papel oposto questiona a correção em múltiplas dimensões.
Memória de Aprendizado O auditor não esquece. Cada erro detectado vira barreira futura. O sistema fica mais rigoroso a cada missão — não começa do zero.

Este formulário fica visível para documentar o payload esperado, mas o botão não dispara nenhuma chamada real até essa decisão ser tomada explicitamente. Não faz push, não faz deploy, não escreve em disco.

Executar Missão — Dry-Run Real (Caminho B, Fase 2a)

Execução real via Vision Agent Local — processo externo, fora do controle direto desta UI. Modo dry-run: simulação em memória, o backend confirma que NUNCA escreve no disco, mas o job roda de verdade fora desta interface.

Apply Fix — Aplicar correção em arquivo real

Aplica a correção diretamente no arquivo do filesystem usando /api/security/apply-fix. Cria backup automático (.bak-s135-*). Confirmação dupla obrigatória.
