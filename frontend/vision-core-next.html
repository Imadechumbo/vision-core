Criar Pull Request

Gerar Patch (Caminho A — seguro, sem escrita automática)

Aplicar Patch Real via Vision Agent Local (Fase 2b)

Aplicação real bloqueada por decisão do usuário, não por falta de mecanismo: o backend já exige agent_secret pareado via /api/agent/register (agent_id sozinho não autentica ninguém). Este formulário fica visível para documentar o payload esperado, mas o botão não dispara nenhuma chamada real até essa decisão ser tomada explicitamente. Não faz push, não faz deploy, não escreve em disco.

Executar Missão — Dry-Run Real (Caminho B, Fase 2a)

Execução real via Vision Agent Local — processo externo, fora do controle direto desta UI. Modo dry-run: simulação em memória, o backend confirma que NUNCA escreve no disco, mas o job roda de verdade fora desta interface.

Apply Fix — Aplicar correção em arquivo real

Aplica a correção diretamente no arquivo do filesystem usando /api/security/apply-fix. Cria backup automático (.bak-s135-*). Confirmação dupla obrigatória.
